SSE State Demo — ArchitectureC4 · Level 2 · Containers

Real-time multi-device job-state broadcasting over FastAPI Server-Sent Events. Numbered flow ①→④ traces one round trip.

Containers (C4 L2)

  • • Single-Page App — React 19 + TS + Vite (in browser)
  • • Web Server / Proxy — nginx, serves SPA + proxies /api
  • • API Application — FastAPI + uvicorn, REST + SSE
  • • Database — SQLite via SQLModel, Alembic-migrated

Real-time broadcast

  • • One global firehose: GET /api/events
  • • snapshot on connect → then live state events
  • • in-process asyncio bus fans out to every subscriber
  • • actions are server-side → broadcast to all devices
  • • scale-out seam: replace bus with Redis pub/sub

Delivery & data

  • • Docker Compose: nginx + FastAPI + volume
  • • nginx tuned for SSE (buffering off, long read timeout)
  • • native EventSourceResponse (FastAPI ≥ 0.135)
  • • job state persisted in SQLite (SQLModel + Alembic)
  • • one-command startup via make up
